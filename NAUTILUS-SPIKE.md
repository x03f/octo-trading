# Спайк NautilusTrader — день 1 (23.07.2026)

Спайк определён как неделя с явным критерием провала. Здесь — результаты первого контакта:
установка + первый сквозной бэктест из нашего озера + инвентаризация трения. Скрипт:
`/opt/octobot/nautilus-venv/spike_backtest.py` (venv изолирован, основной не тронут).

## ✅ Что подтвердилось
- **Ставится на Python 3.13 без Docker** — `nautilus_trader 1.230.0`, колесо с PyPI, ~14 зависимостей.
- **Сквозной бэктест работает и быстрый** — 730 баров BTC/USDT из нашего parquet → движок →
  68 заполнений за **1.2 с** (полный скрипт 3.2 с). Данные грузятся за 0.06 с.
- Готовые инструменты (`TestInstrumentProvider`) снимают ручную возню с точностью для спайка.

## ⚠️ Трение (ровно то, о чём предупреждало исследование про Beta-churn)
Три отдельные поломки в САМОМ ПРОСТОМ бэктесте, каждая — измеренная цена владения:

1. **Штатный `BarDataWrangler` несовместим с pandas 3.0.** Nautilus 1.230 сам тянет pandas 3.0,
   а под его copy-on-write `.values` возвращает read-only буфер, который Cython-загрузчик писать
   не может (`ValueError: buffer source array is read-only`). CoW в pandas 3.0 отключить нельзя
   (опция стала no-op). Обход — ручная сборка баров (лишний код + ловушки точности).
   **Это внутренняя несогласованность релиза: загрузчик ломается о собственную зависимость.**
2. **Строгость к NaN.** Наш движок терпит пропуски через маску; Nautilus падает на первом NaN.
   Для BTC не проблема, но озеро содержит NaN для до-листинговых баров → мульти-монетная
   загрузка потребует пер-монетной обрезки истории.
3. **CASH-счёт не умеет шортить** (`no BTC to deduct`). Наши стратегии лонг+шорт (Сквиз) требуют
   MARGIN-счёта, а это тянет конфиг маржи/плеча/ликвидации — ещё слой настройки.

## ✅ (б) РЕШЕНО: порт Donchian совпал с движком на 100%
`donchian_nautilus.py` — S4 «Черепаха» портирована в Nautilus `Strategy` (event-driven on_bar,
каналы по закрытым барам, лонг+шорт). Прогон на BTC 1d (730 баров) против нашего
`engine/strategies/turtle.py`:
- **Nautilus 43 сигнала, наш движок 43 сигнала.**
- **Совпадение направлений по барам: 39/39 = 100%** (BUY/SELL/FLAT точно там же).
- Прогон 0.8с.

Это снимает главный риск порта: платформа не «примерно похоже считает», а воспроизводит нашу
логику точно. Значит стратегии МОЖНО дублировать на Nautilus без расхождения сигнала.
Оговорка: сверены ТОЧКИ сделок (сигнал), не полный P&L (у Nautilus своя модель исполнения/костов) —
но именно совпадение сигнала было под вопросом, и оно подтверждено.

## Честный промежуточный вывод
Платформа реальна, быстра и ставится без Docker — это подтверждает главное обещание Nautilus
(много инструментов/стратегий в одном процессе, честные плечо/фандинг). **Но подтвердился и
главный риск: Beta-churn не абстрактный.** Три поломки в тривиальном бэктесте, одна — прямая
несовместимость релиза с его же pandas. Версию придётся жёстко пинить и жить без багфиксов,
либо регулярно чинить свой код под их breaking changes (34/29/12/34 пункта в последних релизах).

**Решение по критерию провала — пока НЕ принято** (спайк = неделя, сделан день 1). Остаток:
порт одной стратегии и сверка сделок (б). Но с учётом методического вывода этой сессии
(доказанного края нет ни у одной стратегии) приоритет спайка снижается: **менять платформу
исполнения, пока нечего исполнять с подтверждённым краем, — преждевременно.** Разумно доделать
спайк ПОСЛЕ того, как форвард-тест или новые стратегии дадут хоть один непредвзятый плюс.
