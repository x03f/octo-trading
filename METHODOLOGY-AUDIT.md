# Аудит методики: чистый трёхчастный тест + survivorship (23.07.2026)

Закрыты два методических долга, найденных backtesting-скиллом. Результат неприятный,
но это самый ценный вывод за сессию — он про настоящую цель (есть ли край), а не про инфраструктуру.

## 🔴 Чистый трёхчастный тест — края НЕТ

Старый `run_full.py` отбирал победителя по OOS-Sharpe → OOS стал валидацией, честного теста
не осталось. Именно поэтому S8 показывал «+0.90»: его выбрали ПО этому числу.

Новый `run_test.py`: TRAIN 50% → VALID 25% (отбор) → TEST 25% (один замер, НЕ для отбора).
Верхний терциль ликвидности (где край вообще есть, по Шагу 0), косты Gate.

| стратегия | TRAIN Sh | VALID Sh | TEST Sh | TEST ret |
|---|---|---|---|---|
| S8 «Сквиз» | +0.24 | **−0.29** | +0.70 | +11% |
| S4 «Черепаха» | +0.39 | **−0.50** | +0.56 | +8% |
| S5 «Маятник» | +0.54 | **+0.43** | **−1.67** | **−37%** |
| S1 «Флюгер» | +0.40 | −0.31 | +0.49 | +6% |

**Что произошло:** победитель по VALID — S5 «Маятник» (+0.43, единственный плюс на валидации).
На честном TEST он **−1.67 Sharpe, −37%**. А те, кто реально отработал TEST (S8 +0.70), были
на VALID отрицательны → их бы не выбрали.

**Вывод прямым текстом:** при честной дисциплине отбора **ни одна стратегия не имеет
устойчивого края.** Ранги полностью переворачиваются между VALID и TEST — это режимная
зависимость, а не край. Прошлые OOS-числа (+0.90/+0.64) были оптимистичны, потому что OOS
использовался для отбора. Инфраструктура работает; доказанного края в ней пока нет.

Это ровно тот результат, ради которого затевалась чистая методика: он честный, и он экономит
реальный капитал. Проект уже дважды так сработал (S5, LTF) — теперь в третий раз, на всём наборе.

## 🟡 Survivorship bias — матрица направления смещения

Озеро собрано из СЕГОДНЯШНЕГО топ-500 CoinGecko: монеты, умершие/делистнутые за 2 года,
отсутствуют. Восстановить их данные нельзя (делистнутые не отдают историю), поэтому честный
шаг — задокументировать НАПРАВЛЕНИЕ смещения по каждому классу стратегий:

| Класс стратегии | Направление смещения | Почему |
|---|---|---|
| Лонговые пробойные (S8, S4) | **завышены** | обнулившиеся монеты (пробой вниз → делистинг) вырезаны |
| Mean-reversion лонг (S5) | **сильно завышены** | «откуп дна», которое на деле ушло в ноль, не наказан |
| Шортовые (S11 «Новичок») | **занижены** (консервативно) | самые прибыльные шорты — по монетам, которых уже нет |
| Carry/funding (S2 «Базис») | слабо завышены | делистнутые перпы с экстремальным фандингом отсутствуют |

Практический смысл: TEST-числа лонговых стратегий выше «выше и без того слабые в реальности.
Нельзя сравнивать по Шарпу лонговые и шортовые напрямую — они смещены в РАЗНЫЕ стороны.

Полная реставрация вселенной (исторический снапшот листингов Binance) — отдельная задача на
несколько дней; пока её ценность ниже, чем у вывода «края нет и так». Отложено с явной пометкой.

## Что это меняет в плане
1. **Не заводить реальные деньги ни на одну текущую стратегию** — честный тест края не показал.
2. Живой бумажный контур продолжает копить форвард-данные — это теперь единственный источник,
   не загрязнённый отбором. Если стратегия покажет плюс НА СВЕЖИХ данных, которых не было при
   разработке, — это будет первым непредвзятым сигналом.
3. Новые стратегии S11/S12 проходят через `run_test.py` (трёхчастный), а не через старый отбор.
4. Порог для «завести деньги» из RISK-FRAMEWORK становится строже: TEST Sharpe устойчиво >0.3
   на схеме, не участвовавшей в отборе, И подтверждение форвардом.
