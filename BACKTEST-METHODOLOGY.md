# Методология бэктеста (честная — учимся на folio и на граблях OctoBot)

> Бэктест — это гипотеза, а не обещание. Красивая кривая вверх почти всегда означает
> ошибку в методике, а не найденный грааль. Задача этого документа — чтобы результат,
> которому мы поверим, был честным.

## 1. Ловушки, которые убивают достоверность (проверять каждую)
- **Look-ahead** — использование данных, недоступных на момент решения (в т.ч. исполнение
  по той же свече, где принят сигнал). Решение по закрытой свече, исполнение — на следующей.
  *(В folio эта проверка перевернула вывод: «обгон BTC» исчезал при лаге в 1 день.)*
- **Overfitting / подгонка параметров** — брать не пик на сетке параметров, а **плато**.
  Если работает ровно `period=20`, а `18` и `22` разваливаются — это удача, не edge.
- **Survivorship bias** — тестировать на монетах, которые ДОЖИЛИ; для альтов включать
  делистнутые/мёртвые. Для фьючерсов — учитывать делистинги перпов.
- **Мало независимых режимов** — крипта прожила мало циклов; один бычий/медвежий тренд ≠
  доказательство. Гонять на 2021-2026 И на 2016-2020 (out-of-sample).
- **Косты занижены** — комиссии + проскальзывание + **funding** (для перпов!). Без них
  частые стратегии «зарабатывают» несуществующие деньги.
- **Корреляции → 1 в крахах** — «диверсифицированные» альты падают вместе именно когда важно.

## 2. Обязательный набор проверок перед доверием числам
1. **Атрибуция по периодам** — откуда пришла доходность? Если весь плюс из одного события —
   базис тонкий.
2. **Сенситивность параметров** — сетка вокруг выбранных значений; ищем плато.
3. **Out-of-sample** — прогон на данных, которых НЕ было при настройке.
4. **Walk-forward** — скользящая настройка/проверка (при достаточной длине истории).
5. **Учёт костов** — комиссии + проскальзывание + funding, worst-case-версия.
6. **Стресс-режимы** — отдельно медвежка, боковик, импульс-крах.
Письменный ЧЕСТНЫЙ вердикт: «устойчиво / везение пути», с реалистичными ожиданиями.

## 3. Технические уроки OctoBot-харнесса (чтобы не наступить снова)
- **Детекция завершения бэктеста обязательна**: тяжёлый прогон (20 пар × 6 мес × 1h) идёт
  10-15 мин; читать отчёт РАНЬШЕ завершения = снимок с середины (наблюдали «market −1%»
  вместо реальных −34% и «0 сделок» у стратегии, которая на деле торговала). Ждать финала
  по статусу/логу, потом читать отчёт ОДИН раз.
- **Метрика `market_profitability` OctoBot** = средний рост монет «старт→сейчас»; если она
  не сходится с независимо посчитанным buy-hold — читаем недосчитанный отчёт (см. выше).
- **Свой независимый бенчмарк** (равновзвешенный buy-hold корзины, посчитанный своим кодом
  через ccxt) — как в folio; не доверять только внутренней метрике движка.
- **Детерминизм**: тот же прогон должен давать тот же результат; расхождение = баг харнесса,
  а не «рынок такой».
- Для сравнения многих стратегий надёжнее **свой бэктест-движок на общих данных** (как в
  folio: единый источник правды, паритет), чем гонять UI-движок в цикле.

## 4. Для ФЬЮЧЕРСНЫХ стратегий дополнительно
- Модель funding (историческая ставка по каждому перпу; в удержании — накопленный funding).
- Модель ликвидации (цена ликвидации по плечу/марже; проверка, что стоп срабатывает раньше).
- Изолированная маржа в модели; комиссии фьючерсов (обычно ниже спота, но funding доминирует).
- Проскальзывание масштабируется с плечом (крупнее номинал → хуже цена).

## 5. Definition of Done для бэктеста стратегии
Бэктест считается «доверяемым», когда: полный период прогнан (не хвост), косты+funding учтены,
пройдены проверки §2, есть независимый бенчмарк, результат воспроизводим, и написан честный
вердикт с ожиданиями и слепыми пятнами. Иначе — это красивая картинка, не результат.
