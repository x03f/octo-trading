# octo-trading — криптотрейдинг: стратегии, честный бэктест, живой контур OctoBot

Всё по крипто-трейдингу живёт здесь, включая работу с OctoBot.
Крипто-ETF **«Галс»** — в отдельном репозитории `crypto-portfolio`, сюда не смешиваем.

## Главный принцип
Красивый бэктест ничего не стоит. Всё построено вокруг одного вопроса: **это настоящий край
или подгонка?** Порог решения объявляется ДО замера, и если он не взят — идея закрывается,
а не «дотюнивается». Так уже похоронены две собственные идеи (`RESULTS-VALIDATION.md`,
`RESULTS-LTF.md`) — это нормальный результат работы, а не провал.

## Философия (5 принципов)
1. **Выживание вперёд доходности.** На плече главное — не взорваться (`RISK-FRAMEWORK.md`).
2. **Edge должен объясняться.** Кто на другой стороне и почему проигрывает нам. Нет ответа —
   нет стратегии, сколько бы бэктест ни рисовал вверх.
3. **Бэктест — гипотеза, не обещание.** Красивая кривая ≈ ошибка методики (`BACKTEST-METHODOLOGY.md`).
4. **Реалистичность.** Комиссии + проскальзывание + funding. Без них бумажная прибыль — фикция.
5. **Робастность > подгонка.** Плато параметров, несколько режимов, out-of-sample.

## Ключевые документы
- **`RESULTS-VALIDATION.md`** — walk-forward/OOS. **Читать первым**: опровергает часть in-sample выводов.
- `RESULTS-SUITE.md` — первый (in-sample) прогон. Числам не верить, это proof-of-work движка.
- `RESULTS-LTF.md` — почему идея низких таймфреймов закрыта.
- `STRATEGY-SUITE.md` — портфель из 6 нот, диверсификация по ИСТОЧНИКУ edge.
- `RISK-FRAMEWORK.md` — риск и плечо. `BACKTEST-METHODOLOGY.md` — правила честности.
- `OCTOBOT-LIVE.md` — живой контур и грабли OctoBot (как заставить кастомный тентакл работать).
- `OCTOBOT-CROSSCHECK.md` — сверка своего движка с бэктестом OctoBot.

## Что установлено фактами (а не надеждой)
- **Соло-стратегии — режимные ставки.** Ни одна не дала плюс в обеих половинах истории.
- **Ансамбль устойчивее нот** — но его вклад в снижение просадки, а не в доходность.
- **Cross-sectional momentum в крипте слаб** — проверено на 316 монетах, совпало с литературой.
- **Край на 15m = премия за неликвидность** — на торгуемых топ-50 после костов уходит в минус.

## Процесс разработки стратегии (воронка)
1. **Тезис** — спека по шаблону, edge и риск. Без этого дальше не идём.
2. **Замер края** — дешёвый event-study ДО постройки: есть ли вообще сигнал больше костов.
3. **Прототип-бэктест** — с костами, на своём движке.
4. **Проверки честности** — атрибуция, сенситивность (плато), OOS, walk-forward, бенчмарк.
5. **Риск-профиль** — по RISK-FRAMEWORK (vol-target, стопы, потолки, ликвид-буфер).
6. **Бумажный форвард** — значимый срок в OctoBot; сверка ожиданий с фактом.
7. **Малый капитал → масштаб по факту**, с записанным стоп-краном.

## Запуск
```bash
python selftest.py            # smoke-тест движка
python run_all.py 1d          # весь портфель + бенчмарки + ансамбль
python run_validate.py 1d     # IS/OOS holdout + чувствительность + walk-forward
python run_full.py 1d         # прогон всех нот на полном озере, отбор по OOS
python edge_scan_liq.py 15m   # замер края с фильтром ликвидности
```
Озеро данных в git не хранится — собирается `collect_lake.py` (резюмируемый).

## Живой контур
Бумажные инстансы OctoBot: **реальная торговля выключена, API-ключи не заведены и не планируются.**
Каждый стартует с $10 000 виртуальных. Плечи ×3 / ×10 / ×100 подняты как **измерение цены плеча**,
а не поиск грааля: на ×100 круговая комиссия съедает 4% маржи, ликвидация при движении ~1%.

Реальные адреса панелей держатся локально (`dashboard/endpoints.local.json`, вне git) —
поддомены намеренно неугадываемые, это часть защиты.

⚠️ Всё в СИМУЛЯЦИИ. Реальные плечевые деньги — отдельный, гораздо более опасный шаг,
который начинается только после прохождения gate из `RISK-FRAMEWORK.md`.
